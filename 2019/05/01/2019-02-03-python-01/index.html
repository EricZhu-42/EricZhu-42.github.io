Python-Requests用法简介

Requests

Requests is an elegant and simple HTTP library for Python, built for human beings.

安装方法

使用命令 pip install requests 安装

使用方法

import requests

  5. 读取响应内容

  • 用 r.text 读取服务器响应的内容

  • 用 r.encoding 来获得推测的文本编码，并可以对其进行修改，改变 r.text 的解析方法

  • 用 r.content 读取二进制的响应内容。例如，可以用返回的二进制数据创建一张图片：

  • 用 r.status_code 检测响应状态码，也可以使用内置的状态码查询对象 requests.codes.ok

  • 用 r.headers 来获取响应头内容。

  • 用 r.json() 获取经内置JSON解码器处理的JSON数据。该方法会返回一个JSON对象。当JSON解析失败时，r.json()会抛出一个异常。需要注意的是，成功调用 r.json() 并不意味着响应的成功。要检查请求是否成功，请使用 r.raise_for_status() 或者检查 r.status_code 是否和你的期望相同。

  1. POST一个多部分编码(Multipart-Encoded)的文件
    注意，请使用二进制模式(binary mode)打开文件，防止 Requests 提供的 Content-Length header 发生错误。

  2. 自定义请求头部

  3. 超时
    使用timeout参数使得对象在经过该参数所指定的秒数后停止响应，以此防止程序永远失去响应。
    注意：timeout 仅对连接过程有效，与响应体的下载无关。

  4. 错误与异常
    遇到网络问题（如：DNS 查询失败、拒绝连接等）时，Requests 会抛出一个 ConnectionError 异常。
    如果 HTTP 请求返回了不成功的状态码， Response.raise_for_status() 会抛出一个 HTTPError 异常。
    若请求超时，则抛出一个 Timeout 异常。
    若请求超过了设定的最大重定向次数，则会抛出一个 TooManyRedirects 异常。

  5. 代理
    如果需要使用代理，你可以通过为任意请求方法提供 proxies 参数来配置单个请求：

proxies = {
“http”: “http://user:pass@10.10.1.10:3128/",
}
